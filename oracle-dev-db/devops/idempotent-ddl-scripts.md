# Idempotent DDL Scripts

## Overview

This project deploys schema changes with **hand-run SQL scripts**, not Liquibase or Flyway as the primary workflow.

Patterns:

| Object type | Idempotent pattern |
|-------------|-------------------|
| Tables | PL/SQL block — create if not exists; optional drop-and-recreate flag |
| Views, procedures, functions, packages, triggers | `CREATE OR REPLACE` |
| Seed / reference data | `MERGE` |
| Indexes, constraints (conditional) | Data dictionary check or exception handling |

File names: **lowercase** with underscores — `orders.sql`, `seed_order_statuses.sql`, `install.sql`.

---

## Master Install Script

Use an ordered `install.sql` that calls scripts in dependency order:

```sql
-- install.sql
WHENEVER SQLERROR EXIT FAILURE ROLLBACK
SET DEFINE ON

-- Pass recreate=1 to drop and rebuild tables (dev only)
DEFINE recreate = 0

@@tables/record_statuses.sql
@@tables/customers.sql
@@tables/orders.sql
@@indexes/orders.sql
@@views/vw_open_orders.sql
@@seeds/record_statuses.sql
@@seeds/order_statuses.sql
@@grants/app_user.sql
```

## Table Scripts: Exists Check with Optional Recreate

Oracle has no `CREATE TABLE IF NOT EXISTS`. Use a data dictionary check. Honor a **`recreate`** substitution variable for development resets.

```sql
-- tables/orders.sql
-- Depends on: record_statuses, customers
-- Prefix: acme_ (from oracle-schema-prefix.md)

DECLARE
    v_count   NUMBER;
    v_recreate VARCHAR2(10) := TRIM('&&recreate.');
BEGIN
    SELECT COUNT(*)
    INTO   v_count
    FROM   user_tables
    WHERE  table_name = 'ACME_ORDERS';  -- dictionary stores unquoted names uppercase

    IF v_count > 0 AND v_recreate IN ('1', 'Y', 'YES', 'TRUE') THEN
        EXECUTE IMMEDIATE 'DROP TABLE acme_orders CASCADE CONSTRAINTS PURGE';
        v_count := 0;
    END IF;

    IF v_count = 0 THEN
        EXECUTE IMMEDIATE q'[
            CREATE TABLE acme_orders (
                order_id       NUMBER GENERATED BY DEFAULT ON NULL AS IDENTITY,
                order_number   VARCHAR2(50)   NOT NULL,
                customer_id    NUMBER         NOT NULL,
                status_code    VARCHAR2(30)   NOT NULL,
                created_by     VARCHAR2(128)  NOT NULL,
                created        TIMESTAMP      DEFAULT SYSTIMESTAMP NOT NULL,
                modified_by    VARCHAR2(128),
                modified       TIMESTAMP,
                CONSTRAINT pk_acme_orders PRIMARY KEY (order_id),
                CONSTRAINT uq_acme_orders_order_number UNIQUE (order_number),
                CONSTRAINT fk_acme_orders_acme_customers
                    FOREIGN KEY (customer_id) REFERENCES acme_customers (customer_id),
                CONSTRAINT fk_acme_orders_acme_order_statuses
                    FOREIGN KEY (status_code) REFERENCES acme_order_statuses (status_code)
            )
        ]';
    END IF;
END;
/
```

**Safety:** Default `recreate = 0`. Document clearly when `recreate=1` is appropriate (local dev, ephemeral CI). Never default to drop in production install paths.

## Index Scripts

Create indexes idempotently:

```sql
-- indexes/orders.sql
DECLARE
    v_count NUMBER;
BEGIN
    SELECT COUNT(*) INTO v_count
    FROM   user_indexes
    WHERE  index_name = 'IDX_ACME_ORDERS_CUSTOMER_ID';

    IF v_count = 0 THEN
        EXECUTE IMMEDIATE '
            CREATE INDEX idx_acme_orders_customer_id
            ON acme_orders (customer_id)';
    END IF;
END;
/
```

When multiple indexes exist on one table, use numeric suffixes per `schema-standards.md`.

## Views and Program Units

Always use `CREATE OR REPLACE`:

```sql
CREATE OR REPLACE VIEW vw_acme_open_orders AS
SELECT o.order_id,
       o.order_number,
       o.status_code
FROM   acme_orders o
WHERE  o.status_code = 'OPEN';
/
```

## Seed Data with MERGE

Reference / lookup data must be safe to re-run:

```sql
-- seeds/order_statuses.sql
MERGE INTO acme_order_statuses t
USING (
    SELECT 'OPEN'      AS status_code, 'Open'      AS status_name FROM DUAL UNION ALL
    SELECT 'SHIPPED'   AS status_code, 'Shipped'   AS status_name FROM DUAL UNION ALL
    SELECT 'CANCELLED' AS status_code, 'Cancelled' AS status_name FROM DUAL
) s
ON (t.status_code = s.status_code)
WHEN MATCHED THEN
    UPDATE SET t.status_name = s.status_name
WHEN NOT MATCHED THEN
    INSERT (status_code, status_name)
    VALUES (s.status_code, s.status_name);
```

For tables with audit columns, include `created_by` / `created` on insert and update `modified_by` / `modified` on match when values change.

## SQL*Plus Substitution Variables

- Use `DEFINE recreate = 0` in `install.sql` for the table rebuild flag.
- Use `WHENEVER SQLERROR EXIT FAILURE ROLLBACK` at the top of install scripts.
- Validate dynamic DDL identifiers with `DBMS_ASSERT` when building from parameters — see `db/agent/idempotency-patterns.md`.

## What Not to Touch

- `apex/` export folders
- `f###.sql` APEX application export files at the repository root

## Oracle Version Notes (19c vs 26ai)

Scripts target 19c. Identity columns and `MERGE` are supported. When moving to 26ai, revisit boolean column types per `schema-standards.md`.
